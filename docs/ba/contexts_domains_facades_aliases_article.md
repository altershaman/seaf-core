# Контексты и домены

Контекст это объявленный участник + верхние -- они видны на своем уровне

Домен - это зона интереса участника

Если два объекта объявлены в одном домене, то они видны друг-другу

Нижний домен видит верхний, видит себя внутри и соседей по вышестоящему домену, (но не их внутренности)

Контекст вычисляется из озера

Изначально два соседних домена изолированы, то есть не видят друг-друга, но при этом они видны в меню. Можно зайти в эти домены и погрузиться в них, потеряв видимость предыдущего домена - это смена контекста.

Верхний контекст видит все в нижнем. Но объявленный контекст изолирует свое содержимое от соседей (теперь они не соседи), то есть сверху видимость осталась (**а вот не факт**), а сбоку ее теперь нет.

Вниз смотрим всегда на один уровень. По бокам видим весь горизонт. Наверх видим только один агрегат.

Контекст Процесса, Капабилити, Продукта определяется по владельцу. Участник - есть сам контекст.


# Facades and aliases

Связи всплывают от фасада к контексту

Фасады объявляются на уровне контекста -- Контекст может выставить свой внутренний объект. Объект может напрямую связываться с другими объектами контекста, либо через алиасы с объектами вне своего контекста

Чужие фасады используются через алиасы

Справочник формируется из максимально референтных алиасов (глобально и из контекста)
